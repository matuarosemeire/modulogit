CONFIGURAÇÃO
Há o projeto em si, o git cria o repositório do projeto, ou seja, repositório é o mesmo projeto no GIT
Para configurar o git depois da instalação, no prompt de comando do window
Quando abre há a seguinte informação C:\Users\ro>
Para configurar e salvar o nome do usuário, digita-se:
config --global user.name "Rosemeire Matua" (enter)
Para e-mail: config --global user.email "matuarosemeire@gmail.com" (enter)
Para o editor: git --global core.editor sub (enter)

Para conferir as informações digita-se

git config user.name, git config user.email

Com git config --list, apresenta todas as informações salvas

REPOSITÓRIO
Em modulogit criamos o repositório em si
Para iniciar o repositório - git init
Com este comando o git cria uma pasta oculta, é só habilitar exibição
cd .git, para acessar a pasta; e dir para exibir o conteudo
Branch são as versões do sistema em si
A versão principal chama-se Master
Quando se faz a cópia do sistema, cria-se um novo branch, ou seja, é possível trabalhar em duas versões do mesmo sistema

COMMIT - é a informação que se passa ao git de tudo que foi feito no sistema
Não é backup, o commit só altera o que foi modificado, com no comentário, as explicações das atualizações
O arquivo README.md ou txt, com instruções e comentários, configuração, são informações descritivas
O comando git status, vai informar o que foi modificado
Com esta informação, deve-se adicinar os arquivos a serem modificados que poder ser por:
git add (nome do arquivo), para adicionar um a um ou
git add -A, para adicionar todos os arquivos
Para verificar alteração, git status

COMMITS
COMMITS, são o comentários, com comandos git e entre aspas o comentário
git commit -m "Primeiro Commit"
git log, exibe a lista de todas as modificações naquele branch Depois de criar pastas e alterações, SE NÃO DER COMMIT, a informação vai ficar local Enquanto não der commit, a informação não está disponível no git Cada commit cria um código

REMOVER ALTERAÇÕES (VOLTAR)
O comando - git log - dos commits é verificação do que foi feito, é o histórico
Cada commit registra um código, é um número grande mas o que importa são o sete primeiros dígitos
Depois do comando git log, será exibido o código e HEAD
HEAD - é o branch que se está
git branch - é outra forma de saber em qual branch está, ou seja
Este comando lista todos os branchs, e o selecionado está com um *
git commit -am - adiciona os arquivos e os comentários
Há três tipos de reset: -soft, -mixed e -hard
--soft, um passo atrás, deixa as alterações sem o commit
--mixed, dois passos atrás volta e não prepara para commitar
--hard, volta completamente para o commit anterior, inclusive deletando arquivos
O RECOMENDADO É O SOFT


Para voltar à versão anterior:
01. copia o código do commit
02. digitar: git reset --soft (código inteiro ou 7 primeiros dígitos)
03. git log para conferir

NOVO BRANCH
O branch inicia o sistema do ponto copiado
git branch (nome do novo branch) - para criar um novo branch
git branch - para conferir
Serão exibidos todos os branchs e o * no selecionado, que está sendo trabalhado

TROCAR DE BRANCH
git checkout (nome do arquivo)
git branch - para conferir

git checkout (nome do arquivo) para voltar para outro arquivo

A partir de então, haverão dois históricos, um histórico de cada branch

Com branchs diferentes - ode-se manter versões diferentes do sistema operando simultaneamente
